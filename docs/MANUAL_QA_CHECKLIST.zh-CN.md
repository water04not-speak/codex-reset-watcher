# Codex Reset Watcher v0.2.1 人工桌面验收清单

**适用版本：** v0.2.1  
**建议环境：** Windows 10/11，真实安装包 + 源码开发构建各跑一轮  
**安全要求：** 截图、日志、备注中不得包含 `auth.json`、token、cookie、`.env`、真实完整路径或真实额度敏感信息。

| # | 验收项 | 操作步骤 | 预期结果 | 是否通过 | 备注 |
|---|--------|----------|----------|----------|------|
| 1 | 安装包启动 | 使用 Release 产物安装或运行便携 exe，启动应用。 | 应用窗口正常出现，无白屏、崩溃或系统级错误。 |  |  |
| 2 | 设置保存 | 打开设置，修改语言、主题、刷新间隔或数据源模式，点击保存。 | 设置弹窗关闭；界面立即反映可见设置；无错误提示。 |  |  |
| 3 | 重启后配置保留 | 完成设置保存后退出应用，再重新启动。 | 上次保存的语言、主题、数据源模式、刷新间隔仍保留。 |  |  |
| 4 | auto/manual/mock 三种模式 | 分别切换自动检测、手动脚本、示例数据模式并保存后刷新。 | 三种模式均能进入对应刷新路径；mock 可稳定出数据；manual 不配置时有明确引导。 |  |  |
| 5 | 真实 Codex-Usage 刷新 | 在手动脚本或自动检测中使用真实 Codex-Usage 数据源，点击立即刷新。 | 7–10 秒级慢刷新期间显示阶段文案与 elapsed time；完成后显示额度数据或明确错误。 |  |  |
| 6 | 连续点击刷新 10 次 | 在主界面快速连续点击刷新按钮 10 次。 | 只有一个刷新任务执行；按钮禁用或忙状态清晰；不会出现并发错误或卡死。 |  |  |
| 7 | 自动刷新与手动刷新冲突 | 将刷新间隔设为最小值，等待自动刷新附近手动点击刷新。 | 忙时不会启动第二个刷新；完成后界面恢复可操作。 |  |  |
| 8 | 测试连接 | 在设置中分别测试手动脚本和自动候选数据源。 | 非刷新状态下返回成功或明确失败原因；刷新状态下提示等待刷新完成，不显示 `exec_failed`。 |  |  |
| 9 | 深色 / 浅色主题 | 在设置中切换深色和浅色主题并保存。 | 主题立即变化；主要文字、按钮、卡片在两种主题下均可读。 |  |  |
| 10 | 性能模式 | 打开设置中的性能模式，保存后滚动主界面和设置弹窗。 | 动画、阴影或模糊效果减少；功能不受影响。 |  |  |
| 11 | 1366×768 | 将窗口调整到 1366×768 或相近尺寸，浏览主界面。 | Header、卡片、时间轴、按钮不重叠；主内容可滚动查看。 |  |  |
| 12 | 1920×1080 | 将窗口调整到 1920×1080 或最大化。 | 内容居中且宽度合理；没有异常空白、错位或拉伸。 |  |  |
| 13 | 125% / 150% 缩放 | 在 Windows 显示设置中分别使用 125% 与 150% 缩放后启动应用。 | 文案不被截断；设置弹窗和主界面可完整操作。 |  |  |
| 14 | 设置弹窗滚动 | 打开设置，在较小窗口或高缩放下滚动弹窗。 | 弹窗主体可滚动；顶部标题和底部操作区可用；背景不跟随滚动。 |  |  |
| 15 | 无数据源空状态 | 清空手动脚本路径或在无可用自动源环境启动。 | 显示空状态引导；可选择示例数据、手动配置或重新检测。 |  |  |
| 16 | 错误提示脱敏 | 使用不存在脚本、错误 Python、超时脚本等触发错误。 | 错误可读；不暴露 token、cookie、完整用户目录或原始 stdout。 |  |  |
| 17 | 日志不含 token | 触发刷新、测试连接、失败路径后检查应用日志。 | 日志仅包含状态、耗时、长度或脱敏错误；不含 token/cookie/auth 内容。 |  |  |
| 18 | NSIS 安装 | 运行 NSIS `.exe` 安装包完成安装。 | 安装成功；开始菜单/安装目录可启动；未签名提示符合预期。 |  |  |
| 19 | MSI 安装策略 | 若需要验证 MSI，仅在管理员 / 企业部署场景下运行 `.msi`；普通非管理员用户不作为 v0.2.1 Release 推荐路径。 | NSIS 与便携 exe 作为普通用户发布验收项；MSI 普通权限失败不阻塞 v0.2.1，后续验证 per-user / enterprise 策略。 |  |  |
| 20 | 卸载 | 分别通过 Windows 设置或安装器卸载应用。 | 应用被移除；卸载过程无错误；用户配置保留/清理策略符合发布说明。 |  |  |
| 21 | 升级覆盖 | 先安装 v0.2.0，再安装 v0.2.1 覆盖升级。 | 升级后应用正常启动；配置保留；新刷新提示和测试连接文案生效。 |  |  |
| 22 | README 截图安全检查 | 如需为 README 或 Release 添加截图，使用 mock 模式并复查图片。 | 截图不含真实额度、用户名、路径、token、cookie、auth、日志或真实数据源细节。 |  |  |
